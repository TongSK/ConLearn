# Adversarial Prompt Injection Detection Using Contrastive Learning

> **Master's FYP — Asia Pacific University**
> A contrastive learning framework for detecting adversarial prompt injection attacks in Large Language Models, evaluated under the Leave-One-Dataset-Out (LODO) protocol.

---

## Overview

Large Language Models (LLMs) deployed in production systems are vulnerable to **prompt injection attacks** — adversarial inputs that hijack the model's behaviour by blending malicious instructions with legitimate user prompts. Existing defences rely on keyword filters or fine-tuned classifiers that memorise surface-level attack patterns and fail when attacks are rephrased or obfuscated.

This project addresses that gap by applying **Supervised Contrastive Learning (SupCon)** to learn an embedding space where benign and injected prompts are geometrically separated — enabling detection of unseen attack variants based on intent, not keywords.

The evaluation follows the **LODO protocol** (Fomin, 2026) — training on three dataset sources and testing on the fourth — to produce honest, cross-distribution generalisation results rather than inflated same-distribution benchmarks.

---

## Research Context

| | |
|---|---|
| **Problem** | LLMs cannot structurally separate instructions from data, making prompt injection a fundamental architectural vulnerability |
| **Gap** | Existing detectors overfit to training distributions and collapse under distribution shift (Fomin, arXiv:2602.14161) |
| **Approach** | Supervised Contrastive Loss on RoBERTa encoder, evaluated under LODO cross-distribution protocol |
| **Contribution** | First application of contrastive learning to prompt injection detection with rigorous cross-distribution evaluation |

## Datasets

Four publicly available, ungated datasets are combined into a single benchmark:

| Source | HuggingFace Path | Rows | Notes |
|---|---|---|---|
| DeepSet | `deepset/prompt-injections` | 546 | Clean binary labels |
| Neuralchemy | `neuralchemy/Prompt-injection-dataset` | ~14K | Rich attack category labels |
| SafeGuard | `xTRam1/safe-guard-prompt-injection` | ~8K | Synthetic attack variants |
| Toxic Chat | `lmsys/toxic-chat` (`toxicchat0124`) | ~5K | Real user conversations |

**Combined:** ~27,500 samples after deduplication and cleaning.
**Schema:** `text, label, source, attack_category`
- `label`: `0` = benign, `1` = injected
- `source`: dataset origin — used for LODO partitioning
- `attack_category`: attack type (e.g. `direct_injection`, `jailbreak`, `encoding`)

## Method

### Architecture

```
Input prompt
     │
     ▼
RoBERTa encoder (roberta-base)
     │  768-dim [CLS] embedding
     ▼
Projection head (768 → 256 → 128, ReLU)
     │  128-dim L2-normalised embedding
     ▼
Supervised Contrastive Loss (SupCon, Khosla et al. 2020)
```

### Training Strategy

| Phase | Epochs | Encoder | LR (encoder) | LR (projection) |
|---|---|---|---|---|
| Warmup | 1 | Frozen | — | 1e-4 |
| Fine-tune | 2–10 | Unfrozen | 2e-5 | 1e-4 |

- **Loss:** Supervised Contrastive Loss (temperature = 0.07)
- **Optimiser:** AdamW with linear warmup + linear decay scheduler
- **Early stopping:** patience = 3 epochs on validation loss
- **Checkpoint:** best encoder weights saved by validation loss

### LODO Evaluation

Four folds, one per dataset source. In each fold:
- **Train:** all samples from the other three sources
- **Test:** all samples from the held-out source

This ensures the model is evaluated on attack distributions it has never seen during training — directly addressing the benchmark shortcut problem identified by Fomin (2026).

## Quick Start

### 1. Prepare the dataset

```bash
python dataset_pipeline.py
# Output: dataset.csv (~27,500 rows)
```

### 2. Verify the data loader (optional)

```bash
python data_loader.py dataset.csv deepset
# Expected: batch shapes (16, 64) for input_ids and attention_mask
```

### 3. Verify the model (optional)

```bash
python model.py
# Expected: embedding shape (8, 128), SupCon loss ~1.9–2.6
```

### 4. Train — single LODO fold

```bash
python train.py --csv dataset.csv --held-out deepset --output results/deepset/
```

### 5. Train — all four LODO folds

```bash
python train.py --csv dataset.csv --held-out deepset     --output results/deepset/
python train.py --csv dataset.csv --held-out neuralchemy  --output results/neuralchemy/
python train.py --csv dataset.csv --held-out safeguard   --output results/safeguard/
python train.py --csv dataset.csv --held-out toxic-chat  --output results/toxic-chat/
```

Each fold produces:
- `results/<fold>/best_model.pt` — saved encoder weights
- `results/<fold>/checkpoint_best.pt` — full best epoch checkpoint for validation/test loss
- `results/<fold>/checkpoint_last.pt` — resume checkpoint
- `results/<fold>/training_log.csv` — loss per epoch for learning curve plots

### Faster local training recipes

For a quick laptop run, use fewer epochs, shorter sequences, balanced sampling,
and resume-safe checkpoints:

```bash
python train.py --csv dataset.csv --held-out deepset --output results/deepset/ \
  --epochs 3 --max-length 48 --batch-size 16 --balanced-sampling --resume
```

For a lighter encoder, replace RoBERTa with DistilRoBERTa:

```bash
python train.py --csv dataset.csv --held-out deepset --output results/deepset-distil/ \
  --model-name distilroberta-base --epochs 3 --max-length 48 \
  --balanced-sampling --resume
```

For lower memory full RoBERTa fine-tuning, keep the bottom transformer layers
frozen after warmup:

```bash
python train.py --csv dataset.csv --held-out deepset --output results/deepset/ \
  --freeze-encoder-layers 6 --balanced-sampling --resume
```

## Key Design Decisions

**Why contrastive learning?**
Fine-tuned classifiers memorise surface-level attack patterns and fail when attacks are rephrased. Contrastive learning trains the model to separate prompt *intent* in embedding space — enabling detection of unseen attack variants without lexical shortcuts.

**Why LODO evaluation?**
Fomin (2026) demonstrated that detectors evaluated on the same dataset family as training achieve AUC >0.99 by exploiting provenance shortcuts, not genuine attack semantics. LODO is the only evaluation protocol that measures true cross-distribution generalisation.

**Why freeze the encoder in epoch 1?**
RoBERTa's pre-trained weights are not suited for contrastive learning out-of-the-box. Warming up the projection head first for one epoch before unfreezing the encoder prevents the encoder's representations from collapsing during the initial noisy gradient updates.
